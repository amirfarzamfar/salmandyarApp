# پیاده‌سازی سیستم ارزیابی برای پرستاران و بیماران

## ۱. پیاده‌سازی سمت سرور (Backend)
*   **بروزرسانی `AssessmentsController`**: افزودن اندپوینت `GET /api/assessments/forms/list/{type}` برای دریافت لیست فرم‌های فعال بر اساس نوع کاربر (پرستار یا سالمند).
*   **بروزرسانی `AssessmentService`**: پیاده‌سازی منطق دریافت لیست فرم‌ها بر اساس `AssessmentType`.

## ۲. سرویس‌ها و کامپوننت‌های مشترک فرانت‌اند (Frontend)
*   **بروزرسانی `assessment.service.ts`**: افزودن متدهای دریافت لیست فرم‌ها و ارسال پاسخ‌های ارزیابی.
*   **ایجاد کامپوننت `AssessmentTaker`**: ساخت یک کامپوننت قابل استفاده مجدد (`src/components/assessments/AssessmentTaker.tsx`) که:
    *   سوالات ارزیابی را به صورت داینامیک نمایش دهد.
    *   ورودی کاربر برای انواع مختلف سوالات (چند گزینه‌ای، متنی و ...) را مدیریت کند.
    *   ارزیابی تکمیل شده را به سمت سرور ارسال کند.

## ۳. پنل خدمات‌دهنده (پرستار)
*   **افزودن ناوبری**: بروزرسانی `NurseBottomNav` برای افزودن لینک "ارزیابی‌ها".
*   **ایجاد صفحه لیست ارزیابی‌ها**: ساخت `src/app/nurse-portal/assessments/page.tsx` برای نمایش لیست آزمون‌های موجود برای پرستاران.
*   **ایجاد صفحه جزئیات ارزیابی**: ساخت `src/app/nurse-portal/assessments/[id]/page.tsx` با استفاده از `AssessmentTaker` برای انجام آزمون توسط پرستار.

## ۴. پنل خدمات‌گیرنده (بیمار/سالمند)
*   **افزودن نقطه دسترسی**: اضافه کردن کارت/بخش "ارزیابی‌ها" به صفحه اصلی `PortalPage` (`src/app/portal/page.tsx`).
*   **ایجاد صفحه لیست ارزیابی‌ها**: ساخت `src/app/portal/assessments/page.tsx` برای نمایش لیست آزمون‌های موجود برای سالمندان/خانواده‌ها.
*   **ایجاد صفحه جزئیات ارزیابی**: ساخت `src/app/portal/assessments/[id]/page.tsx` با استفاده از `AssessmentTaker`.

## ۵. تنظیمات پنل ادمین
*   **بررسی و تایید**: اطمینان از اینکه در `AssessmentFormBuilder` نوع ارزیابی (`AssessmentType`) به درستی ذخیره می‌شود تا آزمون‌ها در پنل‌های مربوطه نمایش داده شوند.
